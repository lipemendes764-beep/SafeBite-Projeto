# Metodologia

## Controle de versão

O projeto utiliza Git como sistema de controle de versão e GitHub como repositório central.

### Branches utilizadas
- main: versão estável
- dev: desenvolvimento
- feature: novas funcionalidades
- testing: validação

### Gestão de código

O fluxo segue o modelo GitHub Flow, com commits frequentes e revisão por pull requests antes da integração.

---

## Planejamento do projeto

### Organização da equipe

- Scrum Master: responsável pela organização geral
- Product Owner: definição de requisitos
- Desenvolvimento: implementação do sistema
- Design: interface e protótipos

---

### Processo

O projeto utiliza metodologia Scrum com sprints curtos e acompanhamento via GitHub Projects.

O quadro Kanban é dividido em:
- To Do
- In Progress
- Done

Isso garante acompanhamento contínuo do progresso.
